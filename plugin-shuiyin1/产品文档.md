# 水印插件 产品文档

> 版本：**v0.2.8** | 适用 NocoBase 2.x

## 1. 产品概述

水印插件为 NocoBase 后台管理系统提供全局页面水印功能。在页面最上层覆盖半透明水印，显示当前登录用户名称，防止截图泄露敏感信息。支持通过插件设置页动态配置水印样式和内容，具备防删除与定时刷新能力。

### 1.1 适用场景

- 企业内部管理系统防截图泄密
- 敏感数据页面增加用户标识水印
- 审计合规要求界面带用户水印

---

## 2. 安装

### 2.1 上传安装

1. 在 NocoBase 后台进入 **插件管理**
2. 点击 **上传插件**，选择 `plugin-shuiyin1-<版本号>.tar.gz` 文件
3. **必须进行系统级重启才能生效**（Docker 部署需 `docker restart`，源码部署需重启 Node 进程）

### 2.2 源码安装

```bash
# 在 NocoBase 应用根目录执行
yarn build
yarn nocobase pm enable @my-project/plugin-shuiyin1
```

重启服务后生效。

## 3. 配置说明

启用插件后，进入后台 **设置 → 水印设置** 页面：

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| 启用水印 | 开启 | 开关 | 总开关：关闭后水印 DOM 移除，配置保留不丢失 |
| 水印文字 | 空 | — | 留空则自动使用当前登录用户昵称 |
| 透明度 | 0.15 | 0.01 ~ 1 | 越小越透明，建议 0.1 ~ 0.3 |
| 字号 | 10 | 8 ~ 72 | 单位 px |
| 排列密度 | 5 | 1 ~ 5 | 1 最稀疏，5 最密集 |
| 显示当前年月日时间 | 开启 | 开关 | 开启后水印追加实时日期时间，每分钟刷新 |

修改后点击 **保存**，页面水印**立即生效**，无需刷新。

## 4. 使用说明

### 4.1 基本使用

1. 管理员启用插件后，所有已登录用户在后台页面可看到水印
2. 水印显示当前登录用户昵称，如未设置昵称则显示用户名或邮箱
3. 水印层设置了 `pointer-events: none`，不影响页面正常操作

### 4.2 自定义水印

1. 进入 **设置 → 水印设置**
2. 在「水印文字」输入框填入想要显示的文字
3. 调整透明度、字号、密度等参数
4. 点击 **保存** 即时生效

### 4.3 临时关闭水印

在设置页将「启用水印」开关关闭后保存，水印立即消失。重新打开开关并保存后恢复。

## 5. 技术特性

| 特性 | 说明 |
|------|------|
| 全局覆盖 | 固定定位覆盖全视口，最高层级（z-index: 999999） |
| 用户标识 | 自动读取当前登录用户昵称/用户名/邮箱 |
| 实时刷新 | 显示时间模式每分钟刷新，配置 30 秒周期检测 |
| 防删除 | MutationObserver 监听 DOM 变化 + 2 秒轮询检测，水印 DOM 被移除后自动重建 |
| 即时生效 | 保存配置后通过自定义事件立即通知水印组件重绘 |
| 双端兼容 | 同时提供 client-v1 和 client-v2 实现 |

## 6. 常见问题

### 6.1 保存后水印没有变化

1. 确认已点击 **保存** 且提示保存成功
2. 打开浏览器控制台，查看是否有 `[shuiyin1] watermark refreshed` 日志
3. 若无可尝试刷新页面

### 6.2 刷新页面后水印恢复默认

1. 打开控制台，查看 `[shuiyin1] parsed settings` 日志内容
2. 若配置为空，检查 `shuiyin1_settings` 数据库表是否有记录
3. 确认插件已构建到最新代码

### 6.3 水印文字显示用户名而非自定义文字

水印文字留空时会回退到当前登录用户名。如需自定义文字请填入内容。

### 6.4 水印影响页面操作

水印层设置了 `pointer-events: none`，不会拦截鼠标事件。如仍有问题请检查其他 CSS 是否覆盖了此属性。

### 6.5 生产环境日志过多

在 `src/common/watermark-core.ts` 中将 `DEBUG` 常量设为 `false`，重新构建插件即可关闭所有 `[shuiyin1]` 调试日志。

---

## 7. 安全建议

- 水印仅在前端渲染，无法阻止高级用户通过开发者工具禁用
- 建议配合 NocoBase 权限控制和审计日志使用，形成多层防护
- 对于高度敏感场景，可考虑在后端对导出文件同样添加水印
