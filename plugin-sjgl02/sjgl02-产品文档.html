1. 产品概述

sjgl02（数据管理） 是 NocoBase 平台的一个全栈插件，提供 Excel 数据导入、导出、任务管理、表级权限控制 四大功能。

1.1 入口

插件有 两个入口：

入口路由包含 Tab说明
设置菜单/settings/sjgl02导入 / 导出 / 任务管理 / 权限管理NocoBase 设置页注册菜单项「数据管理」，完整功能入口
区块添加v2 页面「添加区块 → 其他区块 → 数据管理」导入 / 导出 / 任务管理以区块形式添加到 v2 页面，不含权限管理 Tab
#功能入口 Tab说明
1Excel 导入导入3 步向导：选表 → 上传文件 + 字段映射 → 预览 + 执行
2Excel 导出导出3 步向导：选表（含全部数据表）→ 选字段/配置 → 执行 + 下载；支持 .xlsx/.zip
3任务管理任务管理所有导入/导出任务的列表、筛选、日志查看、下载文件、取消
4表级权限权限管理按用户/角色为每张表配置导入/导出权限、字段范围、导入模式等

3. 导入功能 ImportTab

3 步向导（Steps 组件）：选表 → 上传文件 + 字段映射 → 预览 + 执行。当前步骤由 currentStep（0/1/2）控制。

3.1 Step 1：选择数据表

元素说明
标题"选择数据表"
说明"请选择要导入数据的目标数据表"
数据表选择器根据权限配置显示有权限的数据表（filterPermission="import"），搜索过滤
「下一步」disabled 状态直到选中表

3.2 Step 2：上传文件 & 字段映射

3.2.1 上传区

FileUploader 拖拽上传组件。校验 .xlsx/.xls/.csv，最大 50MB。通过标准 attachments:create API 上传文件，成功后自动调用 sjgl02Import:uploadParse 解析并显示 Sheet 列表和表头列名和前 10 行预览数据。

3.2.2 上传后配置区

区域说明
Sheet 选择下拉 Select，切换后自动重新解析表头
表头行号InputNumber 1-100，默认 1，修改后自动刷新；可点击「刷新」按钮手动重新解析
预览表头按钮📋 按钮，点击弹 Modal 显示当前 Sheet 的表头列及前 10 行数据预览；Modal 顶部显示 Sheet 名称、表头行号、数据总行数
导入模式Select：「新增(insert)」「更新(update)」「新增+更新(upsert)」，默认从权限配置读取
唯一值字段仅在 update/upsert 模式下显示。多选组合字段，已选字段在下拉中灰色禁用标注「已选择」
字段映射表见 3.2.3

3.2.3 字段映射表（核心交互）

列顺序：Excel列 → 映射方式（只读） → → 工作表字段

映射方式自动推导规则：

Excel列操作映射方式显示
选了一个 Excel 列名Excel列
选了「自定义内容」Excel列单元格变为输入框，显示 固定值
选了「未选择（忽略）」忽略

去重联动：已选的 Excel 列名在其他行的下拉中自动灰色禁用。选择「未选择（忽略）」后释放该列。

「自动匹配」按钮：按名称自动对应 Excel 列与工作表字段。

标题栏统计：Excel列标题显示「共N/已用M/剩余K」；工作表字段标题显示「共N/已映射M/忽略K」。

3.3 Step 3：预览 & 执行

区域说明
统计卡片预计导入行数、错误行数、导入模式、Sheet 名称
唯一值提示update/upsert 模式下显示蓝色提示条，列出唯一值字段
预览表格前 10 行转换后数据，横向滚动
错误 AlerterrorRows > 0 时显示红色 Alert
「执行导入」弹确认框 → 提交任务 → 成功后 2 秒自动重置到 Step 1
⚠️ 导入事务规则：导入操作在单一数据库事务中执行，任一数据行失败则整批全部回滚，不写入任何数据。不会存在部分成功的情况。

4. 导出功能 ExportTab

3 步向导：选表 → 选字段 & 配置 → 执行导出。支持两种模式：单表导出和全部数据表导出。

4.1 Step 1：选择数据表

下拉菜单第一个选项为 📦 全部数据表（含系统表）（蓝色加粗分隔），其余为具体数据表。选「全部数据表」后 Step 2 标题自动切换为「选择数据范围」。

4.2 Step 2：配置

4.2.1 单表模式

配置项说明
字段选择复选框分组：常规字段 / 关联字段(紫色) / 附件字段(青色)，支持全选，显示「已选: N/M」
关联字段显示模式每个关联字段可选「显示值」/「仅ID」
关联数据 SheetSwitch 开关，开启后多选下拉选择关联表
数据范围「全部数据」/「自定义条件」，选自定义条件展开 FilterConditionBuilder（多行 AND 条件，操作符按字段类型动态切换）
高级选项文件名模板（默认 {表名}_{日期}.xlsx）+ 包含附件文件开关（勾选后，若表中含 attachment 类型字段，将生成 .zip 压缩包，内含 Excel 文件和附件文件夹）

4.2.2 全部数据表模式

顶部蓝色警告：⚠️ 将导出全部 8 张表（含系统表），数据量较大时可能耗时较长。

显示 8 张表的标签列表（含附件标注），文件名模板默认 全部数据表_{日期}.zip。

4.3 Step 3：执行导出

模式统计卡片文件格式
单表选择字段数、预计行数、数据范围、关联Sheet.xlsx（含附件则 .zip）
全部表导出表数量(8)、预计总行数、导出范围(含系统表)、生成文件(.zip)

轮询进度：提交后每 2 秒轮询真实进度（通过 sjgl02Export:progress API），完成后显示下载按钮。

确认弹窗信息：单表显示「生成 .xlsx 文件」；全部表显示「生成 .zip 压缩包（内含 N 个 .xlsx 文件及附件）」。

4.4 文件格式规则

  • 单表 + 无附件 → .xlsx
  • 单表 + 含附件 → .zip（内含 .xlsx + 附件文件夹）
  • 全部数据表 → .zip（内含每张表的独立 .xlsx 文件 + 附件）

4.5 附件导出（v1.0.29+）

勾选「包含附件文件」后，服务端自动识别表中 type='belongsToMany' 且 interface='attachment' 的字段，将附件文件打包进 ZIP。

ZIP 内容说明
{表名}_{日期}.xlsx数据表 Excel，附件列显示为系统文件名（逗号分隔），如 照片_abc123.jpg, 文档_def456.pdf
{附件字段显示名}/按字段分文件夹，内含原始附件文件，文件名与 Excel 中显示的一致
对应关系：Excel 附件列显示的每个文件名，都能在对应字段文件夹中找到同名文件，确保可追溯。

前端内存中维护每个用户/角色的权限配置数据（permStore 对象）。点击左侧切换用户时右侧实时重渲染。卡片上的导入/导出开关点击后立即更新内存并触发重新渲染。

6.6 预置权限数据

系统预设一个管理员角色，拥有全部数据表的全部权限（导入+导出）。

目标拥有的表权限
管理员(角色)全部数据表：导入+导出全开，可导入/可导出字段为空（全部允许），导入模式 insert

7. 通用组件 Components

7.1 TableSelector

搜索过滤的数据表下拉选择器。支持 filterPermission='import'|'export' 按权限过滤（根据权限配置显示有权限的表）。调用 API sjgl02Permissions:tables 获取表列表。

7.2 FileUploader

拖拽上传 Excel。校验 .xlsx/.xls/.csv，最大 50MB。通过 POST attachments:create 上传文件，获取 fileId 后调用 POST sjgl02Import:uploadParse 解析。

7.3 FieldCheckboxGroup

按类型分组展示字段复选框（常规/关联/附件），支持全选。

7.4 FilterConditionBuilder

多行 AND 筛选条件。每行：字段选择 → 操作符（按字段类型动态：字符串有包含/开头等，数字有大于/小于等，日期有早于/晚于） → 值输入 → 删除按钮。支持添加/删除行。

10. 错误处理与边界情况

10.1 通用

  • 所有 API 请求失败 → message.error 提示
  • 操作确认（导入/导出/取消/删除）→ Modal.confirm
  • 任务进度轮询超时 → clearInterval + 提示失败

10.2 导入

  • 不支持格式/超 50MB → Upload.LIST_IGNORE + 提示
  • 必填字段未映射 → 后端校验返回错误
  • 关联字段匹配不到 → 记录行级错误日志
  • 附件下载失败 → 记录行级错误日志
  • 同一 Excel 列不可重复选择（下拉自动禁用）
  • 事务回滚：导入在单一事务中执行，任一数据行失败则整批全部回滚，不写入任何数据

10.3 导出

  • 全部数据表导出前有警告提示
  • 文件名模板支持 {表名} {日期} 占位符

10.4 任务

  • 已完成/失败任务不可取消
  • 导出任务完成后才可下载
  • 未完成的导出任务在日志抽屉中不显示下载

10.5 权限

  • 无权限的表在选择时不可见
  • 权限 API 不可用降级显示所有表
  • 可导入/可导出字段为空数组 = 全部允许

11.3 关键实现细节

  • 动态数据表：表选择器从 sjgl02Permissions:tables API 获取数据库实际表（当前 106 张），非硬编码
  • 动态字段：选表后自动调用 tableFields API 加载该表真实字段，按类型分组（常规/关联/附件）
  • 懒加载 Tab：LazyTab 组件按需异步加载 tab 内容模块，切换时销毁重建，解决状态残留问题
  • 文件校验：beforeUpload 校验 .xlsx/.xls/.csv + ≤50MB，不合法直接 LIST_IGNORE
  • 文件名模板：全表导出和单表导出均支持自定义文件名，支持 {表名} {日期} 占位符
  • 防重入：导出执行中按钮 loading + disabled，完成后才可再次操作
  • 服务端兼容：tables / fields API 适配 NocoBase ctx.db.collections Map 结构
  • v1 认证：v1 客户端通过 this.app.apiClient.request() 调用 API，自动注入 JWT token，无需手动管理认证头
  • 无硬编码：所有表列表、字段列表、用户/角色列表均从 API 动态获取，无任何 mock 数据
  • Sheet 切换联动：切换 Sheet 名称或修改表头行号时，自动调用 uploadParse 重新解析并更新表头列表、字段映射和预览元数据
  • 预览表头弹框：点击📋按钮弹出 Modal，展示当前 Sheet 的完整表头列名 + 前 10 行表格预览，顶部概览 Sheet 名称、表头行号和数据总行数
  • 附件导出：识别 attachment 类型字段（belongsToMany + interface='attachment'），查询时 appends 加载关联附件数据；Excel 中附件列显示系统文件名（逗号分隔）；用 archiver 将 .xlsx 与附件文件打包为 ZIP，附件按字段名分文件夹存放，文件名与 Excel 列一致
  • 日期格式统一：导出时 date/datetime/datetimeTz/unixTimestamp 四类字段统一为本地时区 YYYY-MM-DD HH:mm:ss；导入时自动将 ISO 格式转为标准日期格式；datetimeNoTz/dateOnly/time 三类字段本身兼容，不改动
  • updatedAt 只读：导入字段映射表中，interface='updatedAt' 的字段显示 🔒 只读标签，不渲染 Excel 列选择器，避免用户映射后值被系统覆盖
